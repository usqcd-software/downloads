The tutorial Material

The practical tutorials are below:
  • Tutorial 1: Measurement with chroma This tutorial presents the main chroma application and goes in detail through a single configuration spectroscopy measurement.
  • Tutorial 2: More measurement with chroma This tutorial takes you through the computation of a pion for an ensemble of 5 8x8x8x16 configurations
  • Tutorial 3: Hacking with Chroma - This tutorial takes you through various programming exercises, from mucking about with XML to invoking Factories and linkage.
  • Tutorial 4: Writing your own inline measurement -- does just as advertised. You go through the anatomy of an inline measurement and have to deal with named objects

Notes

Code Versions

Like all codes chroma is evolutionary. I have carried out the tutorial exercises here using

  • QDP tagged version qdp1-20-1
  • Chroma tagged version chroma2-15-2

We try to keep versions up to date but even looking at the output in this tutorial you will find that some output XML quotes the QDP++ version as 1.19.0 rather than 1.20.1. Sorry about that

We are anticipating changes to the API in version 3 of chroma. This may break some of the exercises (Tutorial 3: Exercises 5 and 6 in particular). Please be aware of this.

Compiling and Installing Chroma

We have begun writing document about compiling and installing chroma. The document can be found here.

It currently covers installation in a non-parallel mode on a linux workstation using GNU g++. However it should be a reasonable springboard. However, do be aware of the information below:

Chroma has grown to be quite a complex piece of software. The current map of software modules required for a full build is shown in the image below:

Fortunately, on a Pentium or other SSE assembly capable GNU/Linux or Cygwin workstations, building is relatively straightforward because the third party gmp and libxml2 libraries are shipped as standard with most linux distributions (or are available as easily installable packages). The autoconf and automake packages are also shipped with most linux (and other UN*X) systems however they are frequently out of date. We currently require autoconf-2.58 and automake-1.9.3.

Also the SSE assembly code needed for efficiency is distributed with the QDP++ and Chroma source. On such a platform there is no need for the BAGEL libraries or QMP.
