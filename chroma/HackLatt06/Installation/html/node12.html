Inlining functions and optimisation

The previous build is not very useful. It leaves on the (default) debugging flags and does not do any inlining, resulting in extremely slow code. We found with gcc better code is produced by setting the following: 

/home/bj/src/qdp++/configure --prefix=/home/bj/install/qdp_scalar_single \
    --enable-parallel-arch=scalar CXXFLAGS="-O2 -finline-limit=50000" CFLAGS="-O2"

This sets optimisation level 2 for both GCC and G++. It also sets the inline limit for the C++ compiler to a large value, allowing a large amount of function inlining and should genereate better code.
