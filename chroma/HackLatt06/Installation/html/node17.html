Other errors

Suppose that autoconf and automake are all OK, and you get other errors. A common error is the following during the running of the configure script: 
Error: compiler cannot create executables

This is almost always because something is misspelled like an option to the C++ eg: -fimline-limit=5000, or --with-libxml= instead of --with-libxml2. You should always inspect the config.log file produced by configure to try and ascertain the source of the error.

Only if you have made absolutely certain that you haven't misspelled anything or forgotten some needed configure argument should you send email to the package maintainer.
