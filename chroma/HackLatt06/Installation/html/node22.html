Notes

Some notes and experiences
  • I find that often, when porting to new architectures, QDP++ is the hardest to compile. It has the most options, and if the compiler can't cope with templates this is the first place you know about it. Barring the issues of BAGEL and GMP, a native build of Chroma 4.9 is straightforward as it requires only an installation directory and the path to the QDP++ installation.
  • The configure scripts in QDP++ and Chroma support the --help command line argument. If you get stuck with the configure switches it maybe worthwhile to run 
    configure --help
  • In the fullness of time, this installation guide will grow. Now that you have got to the basics of the install process, and configuring, making and installing you can also try using the example installation scripts you can find in the qdp++/install_scripts and chroma/install_scripts directory of the source packages. These directories often contain README scripts too which may or may not prove enlightening.

Have fun!
