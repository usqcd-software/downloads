Source, Build and Install Directories

Generally, in order for a package to be usable by another package it needs to be installed. Installation can be just about anywhere one has UNIX priviliges to write to. Furthermore, in order for one to be able to build from pristine sources most packages can be compiled in a different directory from the source code. This leads to the concept of having 3 directories per package:
The source directory
- this is the directory which is pulled from CVS, or which results upon untarring a package
The build directory
is the directory in which the code is compiled. It may or may not be distinct from the source directory.
The install directory
is where the software is installed. It can be usually specified with the --prefix=<installdir> command lne argument to configure. If no --prefix argument is given, configure defaults to/usr/local.

Throughout these notes we will always endevour to build in a separate directory from the source.

Let us assume that you have checked out qdp++ to a directory /home/bj/src/qdp++, or you have untarred a qdp++ tarball and moved it to /home/bj/src/qdp++. This directory is your source directory.

Now in order to build qdp++ you would make another directory lets say /home/bj/build/qdp++ and then in that directory you could execute

../../src/qdp++/configure --prefix=/home/bj/install/qdp++

then /home/bj/build/qdp++ is your build directory and /home/bj/install/qdp++ is the install directory. (The install directory will be created automatically if it doesn't exist when you type make install)
